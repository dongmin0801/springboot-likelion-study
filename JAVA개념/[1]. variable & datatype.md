JAVA

[1] Java의 변수 & 자료형

1️⃣ 변수선언의 규칙

1. 1개의 변수에는 1개의 값만 할당 가능함
2. 변수에 저장된 값 -> 재할당을 통해 변경이 가능하다
3. 값의 형태에 맞는 자료형을 사용해야 한다
4. 변수명은 소문자로 시작 (관습)
5. 대소문자 구분, 공백 포함 안됨, 숫자로 시작 불가
6. 예약어는 사용 불가

2️⃣ 자료형 (자료형은 기본형, 참조형으로 나뉜다.)

- 기본형

| 정수형 | 실수형 | 문자형 | 논리형 |
| --- | --- | --- | --- |
| byte  | double | char (10진법과 16진수로 char로 출력하면 ASCII code와 유니코드에 따라 출력된다) | boolean (Java에서는 1과 0을 사용하지 못한다) |
| short | float |  |  |
| int  |  |  |  |
| long |  |  |  |
- 참조형 (메모리에 변수의 데이터가 쓰이는 방법이 기본형과 다르다.)

참조형 데이터의 값 → heap 메모리

변수에 대입되는 값 → heap 메모리 영역의 주소값

ex) String a = new String(”Hello”);

a는 “Hello”라는 문자열이 heap 메모리의 어디에 있는지에 대한 주소만 가지고 있는것이지 값을 가지고 있는건 아니다.

⭐ new연산자를 사용하면  새로운 heap메모리를 사용하지만, 리터럴 방식을 사용하면 heap메모리에 동일한 값이 있는지 찾고 만약 있으면 기존 값을 참조한다.

3️⃣ String

문자열 병합

- + 연산자
    
    String str1 = “Hello ”;
    
    String str2 = “World”;
    
    System.out.println(”str1 + “ ” str2”);  —> “Hello World” 
    
- StringBuilder (속도가 빠르고 메모리를 조금 더 효율적으로 사용할 수 있음)
    
    StringBuilder str = new StringBuilder(”Hello,”);
    
    str.append(” ”);
    
    str.append(”World”);  
    
    System.out.println(str.toString()); —> “Hello, World”
    

문자열 슬라이스

- substring()
    
    String str = “이름: 홍길동”;
    
    String str_name = str.substring(4,7);
    
    System.out.printlf(”str_name”); —> 홍길동
    

문자열 대소문자 변환

- toUpperCase  /  to LowerCase
    
    String str1 = “abc”;
    
    String str2 = “ABC”;
    
    System.out.println(”%s %s”, str1.toUpperCase, str2.toLowerCase); —> “ABC abc”
    
    💡 .equals ⇒ 문자열이 같은지 비교 (대소문자 구분)
    
          .equalsIgnoreCase ⇒ 대소문자 상관없이 문자열이 같은지 비교
    

공백제거

- 양쪽 끝 공백 제거
    
    String str = “      Hello            ”;
    
    str = str.trim();
    
    System.out.printlf(str); —> “Hello”
    
- 문자열 중간 공백 제거
    
    String str = “    He  ll o  ”;
    
    str = str.replace(” “, “”);
    
    System.out.printlf(str); —> “Hello”
    

4️⃣ 형 변환 

자동 형 변환 → 크기가 작은 자료형에서 큰 자료형으로 형 변환 하는 것

강제 형 변환  → 크기가 큰 자료형에서 작은 자료형으로 형 변환을 할 때 , 

                    형 변환 할 수 있는 크기이면 (자료형) 으로 형 변환 할 수 있다. 

            만약 크기가 넘어간다면 오버플로우 발생.

⭐ 자료형의 크기 → byte < short < int < long < float < double
